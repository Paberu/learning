# 1.1. Какие могут быть классы/объекты, придуманные разработчиками, в программах, которые я использую.

## 1) HoMM3: HotA.

Класс Героя (class Hero). В него входят массив хранимых артефактов, массив носимых артефактов, коллекция параметров героя (атака, защита, сила магии, знания, мораль, удача), целоцисленное значение накопленного опыта, коллекция вторичных навыков героя.

Класс Артефакта (class Artifact). В него входят целочисленные значения изменяемых параметров (например, int attack = 6) или коллекция из одной или нескольких пар ключ-значение (если артефакт повышает сразу несколько характеристик).

Класс Существо (class Unit). В него входит коллекция параметров существа, флаги летающего, атакующего или телепортирующегося существа, прочие уникальные флаги и список особенностей данного юнита.

## 2) Google Chrome.

Класс Вкладка (class Tab). В него входят строковое значение адреса (если есть) и закэшированный массив байт - результат предыдущего ответа от веб-сервера.

Класс Закладка (class Bookmark). В него входят строковое значение адреса и путь к графическому файлу (картинке, помогающей визуально отличать закладку от прочих).

Класс История (class History). В него входят коллекция ключ-значение дата-массив посещённых ссылок. Кстати, посещённые ссылки - это встроеный класс (class VisitedURL).
В классе посещённые ссылки есть время загрузки информации, путь к графическому файлу-картинке, как у Закладки, и строковое значение ссылки.
